QKV Projections

Each token embedding is linearly projected into three vectors: Q Query — what am I looking for? K Key — what do I contain? V Value — what do I offer?

Q = X·WQ    K = X·WK    V = X·WV
Attention Scores (Softmax)

The Query token's Q vector is dot-producted with every token's K vector, scaled by √dₖ, then passed through softmax to get probabilities.

Attention(Q,K,V) = softmax( QKᵀ / √dₖ)·V

Weighted Output Vectors

Each token's output is the weighted sum of all Value vectors, weighted by the attention scores. The output has the same dimension as the input embedding.

Query
Represents what the current token is searching for in the sequence. Acts as the "question".
Key
Represents what information each token "advertises". The Q·Kᵀ dot product measures compatibility.
Value
The actual content that gets aggregated. High-attention tokens contribute more to the output.
Scale √dₖ
Prevents dot products from growing too large in high-dimensional spaces, keeping softmax gradients stable.
